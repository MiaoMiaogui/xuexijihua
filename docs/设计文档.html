四、关键设计说明

1. 信息架构：以首页为中枢，五大 Tab 覆盖 MVP 到高阶能力；打卡为底部中央 FAB，符合“≤1 次点击”原则。
2. 角色差异化：学生端最全；家长/教师端为只读协同视图，仅暴露汇总指标与异常提醒，保护隐私（方案 2.5）。
3. 学习科学闭环：计划向导(目标倒推) → 日历执行 → 打卡 → 统计反馈 → 错题再练。
4. 激励与防 burnout：连续打卡、徽章、热力图正向反馈；番茄钟与休息提醒防止虚假勤奋（方案 2.3）。
5. 性能：高频读走 Redis 缓存，写后主动失效，降低 MySQL 压力（方案 2.4）。
